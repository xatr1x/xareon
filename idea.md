

# Xareon

## Бачення проекту

**Xareon** — це персональний настільний ігровий журнал, а не каталог ігор. Головна ідея — зберігати спогади, думки, досягнення, скріншоти та особисту історію гравця протягом багатьох років. Xareon не призначений для збору інформації про всі ігри світу, а для фіксації вашого унікального досвіду та вражень від ігор.

## Технологічний стек

- [Tauri v2](https://tauri.app/) — настільна оболонка.
- Rust (бекенд).
- TypeScript.
- HTML.
- CSS.
- Vanilla JavaScript (або TypeScript на фронтенді).
- SQLite (використати найбільш відповідну бібліотеку для Tauri/Rust, наприклад, `rusqlite` або альтернативу, яка забезпечує просте використання та міграції).
- **Без** React, Vue, Angular, Svelte чи інших фронтенд-фреймворків.

## Архітектурні вимоги

- Чиста архітектура (Clean Architecture).
- Розділення на рівні: UI, application/services, repositories, database, domain models.
- Відсутність бізнес-логіки у UI.
- Модульний код.
- Легка розширюваність.

## Вимоги до UI

- Мінімалізм.
- Темна тема.
- Відчуття нативного десктопного застосунку.
- Адаптивне змінення розміру вікна.
- Відсутність зайвих анімацій.

## Початковий функціонал

- CRUD для ігор.
    - Поля: назва, жанр, платформа, розробник, видавець, рік релізу, дата початку, дата завершення, статус, оцінка, обкладинка.
- Статуси ігор: Заплановано, Граю, Призупинено, Завершено, Завершено на 100%, Кинуто.
- Особисті щоденникові записи з датою/часом.
- Перегляд у вигляді хронології (timeline).
- Особисті теги.
- Галерея скріншотів.
- Сторінка статистики (завершено ігор, середня оцінка, граю, кинуто, всього записів у щоденнику тощо).

## Вимоги до системи досягнень

- Універсальна система.
- Досягнення для кожної гри.
- Поля досягнення: назва, опис, прапорець "відкрито", дата відкриття, прогрес (поточне/ціль), необов’язкова категорія, необов’язковий прапорець "приховано".
- Можливість створювати власні досягнення, не пов’язані з офіційними досягненнями гри.

## Вимоги до бази даних

- SQLite.
- Коректні міграції.
- Зовнішні ключі (foreign keys).
- Рівень репозиторіїв для роботи з даними.
- Відсутність сирого SQL, розкиданого по проекту.

## Вимоги до якості коду

- Суворий TypeScript.
- Сильна типізація.
- Коментарі лише там, де це дійсно корисно.
- Послідовне найменування.
- Відсутність дубльованого коду.

## Можливості для майбутнього розширення

- Імпорт бібліотеки Steam.
- Резервні копії та відновлення.
- Хмарна синхронізація.
- Плагіни.
- Трекінг часу гри.
- Пошук та фільтрація.
- Експорт/імпорт даних.

## Інструкції для AI-агента

- Повністю створити проект згідно цієї специфікації.
- Якщо технічне рішення не зазначене — обирати те, що максимально підвищує підтримуваність та простоту.
- Мінімум залежностей.
- Не ускладнювати архітектуру без потреби.
- Створити структуру проекту, придатну для продакшн-розгортання.

## Вимоги до AGENTS.md

Створити файл `AGENTS.md` у корені репозиторію.

Файл **AGENTS.md** повинен містити:
- Призначення проекту.
- Огляд архітектури.
- Структуру директорій.
- Технологічний стек.
- Як зібрати та запустити проект.
- Опис бази даних.
- Опис модулів.
- Використані конвенції.
- Правила розробки.
- Як додавати нові функції.
- Важливі архітектурні рішення.
- Відомі обмеження.
- Дорожню карту розвитку.

**Після кожної модифікації проекту AI-агент зобов’язаний оновлювати AGENTS.md так, щоб він завжди відображав поточну реалізацію. AGENTS.md є єдиним джерелом правди для майбутніх AI-агентів, які працюватимуть із цим репозиторієм.**
